# 用户使用手册（本地运行）

本手册用于在 Windows 本地把本项目“前端 + 后端 + 数据库”跑起来。

## 1. 准备环境

- JDK 17
- Maven
- Node.js + npm
- MySQL 8.x
- Redis

## 2. 初始化数据库

1) 确保 MySQL 已启动
2) 执行脚本 `db/big_event_init.sql`

你可以用 Navicat/Workbench 执行，也可以命令行执行：

`mysql -uroot -p < db/big_event_init.sql`

## 3. 配置本地连接信息（避免提交密码）

1) 复制 `.env.local.example` 为 `.env.local`
2) 修改 `.env.local` 中的 `DB_PASSWORD`

注意：`.env.local` 已在 `.gitignore` 中忽略，适合放本机密码，不要提交到 GitHub。

## 4. 安装依赖并启动

前端首次运行需要先安装依赖：

- `cd frontend && npm i`

启动（推荐）：

- 双击 `dev.cmd`

或 PowerShell：

- `powershell -ExecutionPolicy Bypass -File .\\dev.ps1`

## 5. 验证

- 前端：`http://localhost:5173/`
- 后端：`http://localhost:8081/`（默认）

## 6. 常见报错排查

- 前端 `ECONNREFUSED`：后端没起来/端口不对；检查后端窗口日志，确认后端端口在监听（默认 `8081`）。
- 后端提示 `Port ... was already in use`：说明端口被占用；`dev.cmd` 会自动尝试 `8081…8090`，你也可以在 `.env.local` 里手动设置 `SERVER_PORT`。
- 后端编译失败并提示 `JCTree...qualid`：请使用 JDK 17（脚本会优先选择本机 JDK 17）。
